## 0. 자동 적용 규칙 (Auto-apply) — 작성자/에이전트가 기계적으로 따를 것

물리적 주소나 방문 고객이 있는 KORVIA 자산(사무실·미술센터·매장)을 다룰 때 **아래를 무조건 실행**한다. (근거·상세는 본문 절 참조.)

1. **[이중 등록 필수]** 물리 위치가 있으면 **네이버 스마트플레이스(한국어·한국인)** + **Google Business Profile(영어·외국인)** 를 **둘 다** 등록·인증한다. 하나만 하지 않는다. → §1, §2, §4-2
2. **[NAP 단일 표기 고정]** 상호(Name)·주소(Address)·전화(Phone)를 **글자 단위로 동일**하게: 홈페이지 `LocalBusiness` JSON-LD ↔ 스마트플레이스 ↔ GBP ↔ 카카오맵. 층·호수·도로명 표기까지 통일. 표기가 갈리면 지도가 "다른 업체"로 인식해 인기도(prominence)가 분산된다. → §4
3. **[카테고리 = 최우선 관련성 신호]** GBP **주 카테고리(primary category)** 와 네이버 **대표 업종**은 랭킹의 단일 최강 신호다. 가장 정확한 1개를 주 카테고리로, 나머지는 보조 카테고리로. 애매하면 실제 매출 핵심 서비스 기준. → §1-2, §2-2
4. **[대표키워드/서비스지역]** 네이버=대표키워드(최대 5개, 지역명+업종 조합) 정확 설정. GBP=서비스 항목·속성(attributes)·서비스지역(SAB면 주소 숨김) 채움. → §1-3, §2-4
5. **[리뷰 정공법]** 리뷰는 랭킹 상위 신호(로컬팩 ~20%)지만 **매크로/유입 조작·대가성 리뷰·영수증 위조는 즉시 제재**. 방식은 오직 "실제 방문/이용 고객에게 QR·링크로 자연 요청 + 전건 응답". 최근성(review recency)이 중요 → **꾸준히**. → §1-5, §2-3
6. **[사진·소식 정기 갱신]** 스마트플레이스 소식/GBP posts를 **주 1회 이상**, 실제 촬영 사진(스톡·AI 금지) 정기 업로드. 방치 프로필은 행동데이터·최신성에서 밀린다. → §1-4, §2-5
7. **[행동데이터가 곧 인기도]** 네이버 2025 로직은 **실사용자 행동(클릭·지도보기·전화·길찾기·체류·재방문)** 을 핵심 신호로 삼는다. 인위적 대량 유입은 "비정상 패턴"으로 역효과. 홈페이지·블로그·SNS에서 플레이스로 자연 유입 경로를 만든다. → §1-6
8. **[영업정보 정확·즉시]** 영업시간·임시휴무·전화·URL 변경은 지도에 **즉시** 반영. 오정보는 이용자 행동(이탈) 악화 + 신뢰 하락. → §1-4, §2-5
9. **[JSON-LD 동기화]** 홈페이지에 `LocalBusiness`(또는 하위 타입) JSON-LD를 넣고 `name/address/telephone/openingHours/geo/sameAs`(플레이스·GBP URL)까지 채워 지도 프로필과 일치시킨다. 상세는 [`SCHEMA_JSONLD_CATALOG.md`](./SCHEMA_JSONLD_CATALOG.md). → §4-1
10. **[검증 후 기재]** 인증 방식(영상/엽서/전화)·메뉴 라벨·카테고리 목록은 발행 전 콘솔 실화면에서 재확인. `⚠` = 비공식/시점의존 값.

# PART 1 — 네이버 스마트플레이스 (한국어·한국인 채널)

> **왜 1순위인가**: 한국 로컬 검색의 사실상 관문은 네이버다. 지도앱·통합검색 상단 **'플레이스' 블록**이 오프라인 업체 유입의 주 경로. 스마트플레이스 등록·유지·노출은 **전부 무료**(광고 별도).

## 1-1. 등록 & 인증

| 단계 | 내용 | 비고 |
|---|---|---|
| 1. 접속 | <https://smartplace.naver.com> → 네이버 로그인 → **[신규 등록]** | 사업자 계정 권장 |
| 2. 업체 검색 | 기존에 이미 노출 중인 업체인지 먼저 검색(사용자가 만든 플레이스가 있을 수 있음) → 있으면 **소유권 주장(내 업체 등록)** | 중복 생성 금지 |
| 3. 업체 정보 | 상호·주소(도로명, 층/호수)·전화·업종·영업시간·상세소개 입력 | NAP 고정 → §4 |
| 4. 인증 | **사업자등록증 업로드**(관리자 권한) + **전화 인증**(업체 대표번호로 발신/ARS) | ⚠사업자 직접 등록·인증한 정보가 노출 우선순위가 더 높은 경향 |
| 5. 승인 | 검수 후 노출. 통상 수 시간~수일 | 카테고리·주소 이상 시 반려 |

> **핵심**: 사장이 **직접** 사업자등록증으로 소유권을 확보한 업체가, 사용자가 임의 생성한 업체보다 신뢰·우선순위가 높다. 이미 존재하는 유령 플레이스가 있으면 반드시 소유권을 흡수한다.

## 1-2. 업종 카테고리 (대표 업종 = 관련성 최강 신호)

- **대표 업종 1개** + 필요 시 세부 업종. 대표 업종이 "이 업체가 무엇인가"를 규정하는 최상위 관련성 신호.
- 실제 핵심 서비스와 정확히 일치시킨다. 유입 노리고 무관한 인기 업종 선택 금지(반려·품질 하락).
- 예: 미술학원/미술센터, 통신/휴대폰, 경영컨설팅 등 네이버 제공 업종 트리에서 가장 근접한 것.

## 1-3. 대표키워드 & 상세정보

- **대표키워드**(최대 5개 ⚠): "지역명 + 업종/서비스" 조합으로 실제 검색되는 형태. 예: `강남 미술심리`, `서초 원어민채용`. 억지 스터핑 금지.
- **상세소개**: 서비스·특장점·이용 안내를 자연어로. 대표키워드를 문맥상 자연 포함.
- **부가정보**: 주차, 예약, 편의시설, 홈페이지·블로그 URL(→ NAP·sameAs 동기화), SNS.

## 1-4. 사진 · 소식 · 예약 · 톡톡

- **사진**: 실제 촬영 사진을 다수·고화질로. 외관/내부/서비스/대표. 최신성 유지(정기 추가). 스톡·AI 이미지 지양.
- **소식(플레이스 소식)**: 새 소식·이벤트·공지를 **주 1회+** 발행 → 최신성·행동데이터·통합검색 노출에 유리.
- **네이버 예약 / 네이버 톡톡**: 예약·문의 전환 채널 연결 시 클릭·전화·문의 등 **행동 신호**가 쌓여 인기도에 직접 기여.
- **영업시간·임시휴무**: 항상 정확·즉시. 오정보는 이탈(부정 행동신호)로 이어진다.

## 1-5. 리뷰 — 방문자 리뷰 / 영수증 리뷰 & 어뷰징 절대 금지

- **방문자 리뷰**: 실제 방문/이용 고객이 작성. 사진·텍스트.
- **영수증 리뷰**: 결제 영수증 인증 기반(실제 방문 검증). 신뢰도 높은 리뷰 유형.
- **정공법만**: 방문 고객에게 **QR/링크로 자연 요청**, 모든 리뷰에 성실히 응답(질문·불만 포함).
- 🔴 **금지·제재 대상**: 대가성/허위 리뷰, 리뷰 매크로·구매, 영수증 위조, 지인 대량 동원. 네이버는 인위적 패턴을 감지해 **노출 제한·순위 하락·플레이스 제재**. 2025 로직에서 "리뷰 수 그 자체"의 비중은 낮아지고 **실사용자 행동데이터**로 무게중심이 이동 → 조작의 실익은 줄고 위험만 커졌다.

## 1-6. 네이버 플레이스 랭킹 요인 (2025~2026 로직)

네이버는 공식 랭킹 공식을 공개하지 않지만, 실무·업계 분석상 세 축으로 수렴한다 ⚠:

1. **관련성(정확도)**: 대표 업종·대표키워드·업체명·상세정보가 검색어와 얼마나 부합하는가. 카테고리·키워드 정합이 1차.
2. **거리**: 검색자 위치(IP·GPS) 또는 질의에 명시된 지역 중심과 업체 위치의 근접도.
3. **인기도/행동데이터**: **실사용자 행동**(검색 후 클릭, 지도 보기, 전화 연결, 길찾기, 체류시간, 재방문 패턴)이 핵심 신호. + 리뷰·저장·소식 활동.

> ⚠ **2025년 알고리즘 전환의 핵심**: "리뷰 수·운영기간" 중심 → **유입의 구조와 비율(자연스러운 실사용자 행동)** 중심으로 이동. 검색량 대비 과도한 유입 급증은 **비정상 패턴**으로 감지되어 노출 제한. 따라서 홈페이지/블로그/SNS/오프라인에서 **꾸준하고 자연스러운** 플레이스 유입 경로를 설계하는 것이 정공법. (출처: 파인애드·아이보스 등 업계 분석 — 비공식, 시점의존.)

## 1-7. 노출 위치

- **네이버 지도(앱/웹)** 검색 결과.
- **통합검색 '플레이스' 블록**: 지역·업종 질의 시 상단 카드.
- 스마트플레이스 정보는 통합검색·지도·모바일 전반에 활용됨(플레이스가 소스 오브 트루스).

# PART 2 — Google Business Profile (구 Google My Business, 영어·외국인 채널)

> **왜 필수인가**: 외국인·해외 이용자는 구글맵을 쓴다. KORVIA의 외국인 대상 사업(원어민 채용, eSIM, 유학)에서 **영어 노출 = GBP**. 관리는 별도 앱이 폐지되고 **Google 검색/지도 내 인라인 편집**으로 통합됨.

## 2-1. 생성 & 인증 (2026: 영상 인증이 기본)

| 단계 | 내용 |
|---|---|
| 1. 생성 | Google 계정 로그인 → <https://business.google.com> 또는 Google 지도/검색에서 업체 검색 → **[비즈니스 프로필 관리/추가]** |
| 2. 기본정보 | 상호·카테고리·주소(또는 서비스지역)·전화·웹사이트 입력(NAP 고정 → §4) |
| 3. 인증 | **영상 인증(video verification)** 이 2026년 대다수 신규 등록의 **기본**. 매장·간판·주변·내부·업무증빙을 한 번에 촬영 |

- ⚠ **엽서(postcard)**: 상당수 카테고리(특히 서비스지역 업체)에서 **폐지/축소**. 노출 빈도 낮음.
- ⚠ **전화/이메일**: Google이 사전 승인한 좁은 카테고리에만 남음.
- 🔴 **2026 인증 강화**: 인증 단계가 복잡해졌고(비즈니스 모델 확인 단계 추가), **반려율이 급증**. 영상 인증 팁: 실제 간판·주소 표지·내부·직원/업무 활동을 **끊김 없이 한 컷**으로, 프로필의 주소/카테고리와 실제가 일치하도록 촬영. 재촬영 요구를 줄이려면 첫 촬영에서 주소 증빙을 명확히.

## 2-2. 카테고리 (주 + 보조)

- **Primary category(주 카테고리)**: **로컬팩/맵 랭킹의 단일 최강 관련성 신호**(2026 Local Search Ranking Factors 설문 1위). 가장 정확한 1개.
- **Additional categories(보조)**: 부가 서비스 표현. 관련성 보강.
- 실제 핵심 사업과 일치. 무관한 인기 카테고리 남용은 정지 위험.

## 2-3. 리뷰 관리 & 응답

- 리뷰는 로컬 랭킹에서 큰 비중(2026 설문상 로컬팩 영향의 **~20%** ⚠). **최근성(recency)** 이 2026에 더 중요 → **꾸준한 유입**이 정체된 프로필을 이긴다.
- 모든 리뷰에 **응답**(감사·문제해결). 응답은 신뢰·전환·부분적 랭킹 신호.
- 🔴 리뷰 게이팅(부정 리뷰 차단)·대가성·가짜 리뷰는 Google 정책 위반 → 삭제·정지.

## 2-4. 서비스지역 · 속성 · 서비스 항목

- **Service Area Business(SAB)**: 매장 없이 방문 서비스면 주소 숨기고 **서비스지역** 지정.
- **Attributes(속성)**: 편의시설·접근성·결제·"여성 소유" 등. 관련성·전환 보강.
- **Services / Products**: 제공 서비스·상품 목록화 → 관련성.

## 2-5. GBP Posts · Q&A · 사진

- **Posts**: 업데이트·이벤트·오퍼를 **정기(주 1회+)** 게시 → 활성 신호·전환.
- **Q&A**: 자주 묻는 질문을 **오너가 선제 등록·답변**(잘못된 답변 방치 금지).
- **사진**: 실제 촬영. 외관/내부/서비스. 정기 추가로 최신성.

## 2-6. 로컬팩 / 구글맵 랭킹 3요인 (Google 공식)

Google 공식 문서 기준 로컬 랭킹은 **relevance · distance · prominence** 3요인:

1. **Relevance(관련성)**: 프로필이 검색어와 얼마나 부합. → **주 카테고리·완성도**가 핵심.
2. **Distance(거리)**: 검색자 위치(기기 위치 또는 질의에 명시된 지역의 중심점)와 업체 거리.
3. **Prominence(인기도/저명성)**: 리뷰·링크·인용(citation)·언급·웹 전반의 평판. 경쟁 도심에서는 **prominence가 승부처**.

> ⚠ **2026 Whitespark Local Search Ranking Factors** 설문(비공식·업계): 로컬팩 영향의 대략 **GBP 신호 ~32% / 리뷰 ~20% / 온페이지 웹사이트 신호 ~15%**. 즉 GBP 최적화 + 리뷰 + 홈페이지 로컬 신호를 함께 관리해야 한다.

## 2-7. 외국인 대상 영어 노출 팁

- 상호/설명/서비스/게시글을 **영어**로(필요 시 한/영 병기). 외국인 질의는 영어 → 영어 콘텐츠가 관련성 확보.
- 홈페이지 영어 페이지에 `LocalBusiness` JSON-LD(영문) + GBP `sameAs` 상호링크.
- 외국인 리뷰(영어)를 자연 유도 → 영어 질의 prominence·관련성 동시 강화.

# PART 4 — NAP · 데이터 일관성 & 이중언어 전략

## 4-1. NAP · JSON-LD ↔ 플레이스 ↔ GBP 동기화

- **N(상호)/A(주소)/P(전화)** 를 **모든 채널에서 글자 단위 동일**하게: 홈페이지 `LocalBusiness` JSON-LD · 스마트플레이스 · GBP · 카카오맵.
  - 주소: 도로명/지번, 층·호수 표기까지 통일(예: "서울특별시 서초구 …로 …, 25층 2517호").
  - 전화: 대표번호 1개(국가코드 표기 통일).
- 홈페이지 `LocalBusiness` JSON-LD 필수 필드: `name`, `address`(PostalAddress), `telephone`, `openingHours`, `geo`(lat/lng), `url`, **`sameAs`**(스마트플레이스 URL·GBP URL·카카오맵 URL). 상세·타입 선택은 [`SCHEMA_JSONLD_CATALOG.md`](./SCHEMA_JSONLD_CATALOG.md).
- 🔴 표기가 채널마다 갈리면 검색엔진이 **동일 업체로 확신하지 못해** citation/prominence가 분산 → 로컬 랭킹 손해. NAP는 **한 번 정하면 전 채널 일괄 수정**.

## 4-2. 이중언어 채널 분담

| 채널 | 언어 | 타깃 | 우선순위 |
|---|---|---|---|
| 네이버 스마트플레이스 | 한국어 | 한국인 | **1순위(국내)** |
| Google Business Profile | 영어(+한글 병기) | 외국인·해외 | **1순위(외국인)** |
| 카카오맵 | 한국어 | 한국인 보완 | 2순위 |
| 애플맵(Business Connect) | 영어 | iPhone 외국인 | 3순위 ⚠가용성 확인 |

- 원칙: **한국어=네이버, 영어=GBP**. 두 프로필의 NAP는 동일하되, **설명·게시글·서비스명은 각 언어로** 현지화.

# PART 5 — 4개 사이트 적용 & 즉시 액션

## 5-1. ACT Art Center (오프라인 미술센터) — 🟢 로컬이 1순위 채널 (최우선·상세)

오프라인 방문형 미술센터이므로 **로컬 검색 = 핵심 유입 채널**. 홈페이지 SEO보다 플레이스가 매출 직결.

**즉시 액션**
1. **스마트플레이스 등록**: 사업자등록증으로 소유권 확보. 대표 업종=미술학원/미술심리/문화센터 중 실제 최근접 1개. 주소·층/호수·전화 정확.
2. **대표키워드**: `지역명+미술`(예: `강남 미술심리`, `서초 아동미술`, `성인 미술클래스`) 실제 검색형으로 최대 5개.
3. **사진**: 센터 외관·작업실·수업 장면·작품 실촬 다수 업로드. AI/스톡 금지.
4. **소식·예약**: 클래스 일정/모집을 소식으로 주 1회+, **네이버 예약** 연결로 예약·전화 행동데이터 축적.
5. **리뷰**: 수강생에게 QR/링크로 방문자·영수증 리뷰 자연 요청, 전건 응답. 절대 조작 금지.
6. **GBP**: 외국인 수강생·거주 외국인 타깃이면 영어 GBP 병행(카테고리=Art center/Art school). 영상 인증.
7. **홈페이지**: `LocalBusiness`(하위: `ArtGallery`/`School` 적절) JSON-LD + `sameAs`(스마트플레이스·GBP). → [`SCHEMA_JSONLD_CATALOG.md`](./SCHEMA_JSONLD_CATALOG.md).
8. **카카오맵**: 매장관리로 등록(보완).

## 5-2. Kimchi (통신/eSIM) — 픽업/매장 유무에 따라 조건부

- Kimchi는 **온라인 eSIM/통신** 중심이라 물리 매장·오프라인 픽업 지점이 **있을 때만** 로컬 등록 대상.
- **오프라인 픽업/매장 있음** → 해당 지점 스마트플레이스 + GBP(카테고리=Telecommunications service provider / Cell phone store) 등록. 외국인 대상이므로 **GBP 영어 노출이 특히 중요**.
- **매장 없음(순수 온라인)** → 로컬 등록 대신 **서비스지역(SAB)** 판단 또는 로컬 제외. 웹/블로그/KIMO는 기존 문서(멀티엔진·AEO/GEO) 트랙으로.

**즉시 액션**: 오프라인 지점 존재 여부 확인 → 있으면 GBP 우선(외국인) + 스마트플레이스, NAP 통일. 없으면 본 문서 대상 아님(스킵).

## 5-3. Korvia / Passome (사무실) — 사무실 로컬 프로필 (낮은 우선순위·정합성 목적)

- B2B 컨설팅/에이전시 사무실은 방문 고객 유입이 크지 않아 로컬 우선순위는 낮지만, **브랜드 검색 시 지도 카드(지식패널)·NAP 정합성**을 위해 등록 권장.
- **Korvia**(원어민 채용·유학, 외국인 접점) → **GBP(영어) 우선**: 카테고리=Employment agency/Consultant. 외국인 파트너·후보자가 브랜드 검색 시 신뢰 있는 지도 카드 확보. 스마트플레이스(한국어)도 병행.
- **Passome**(㈜피에이에스컨설팅, 서초 2519호) → 스마트플레이스=경영컨설팅. 신설 법인 브랜드 저명성 확보용. NAP=사업자등록 주소와 동일(2519호까지).
- 같은 건물(서초 2517/2518=Korvia, 2519=Passome)이므로 **호수까지 구분한 정확한 주소**로 각 프로필을 분리, 상호 혼동 방지.

**즉시 액션**: 각 법인 GBP/스마트플레이스 생성 → 사업자등록 주소·대표번호·홈페이지로 NAP 고정 → 홈페이지 `LocalBusiness`(Korvia=`EmploymentAgency`, Passome=`ProfessionalService`) JSON-LD `sameAs` 연결.
